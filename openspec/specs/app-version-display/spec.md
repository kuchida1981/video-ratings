# App Version Display Spec

## Purpose

アプリケーションの現在のバージョン（Gitタグベース）をサイドバーフッターと設定画面に表示する機能を提供する。
開発環境では `git describe --tags --always` による動的なバージョン生成、リリースビルドでは CI/CD 経由のタグ名埋め込みにより、どの環境でも正確なバージョンを確認できる。

## Requirements

### Requirement: アプリケーション全体のバージョン情報が画面のフッターに表示されること
システムは、動作中のアプリケーションのバージョン番号（Gitタグ）を、左側のナビゲーションサイドバーの底部に表示しなければならない（SHALL）。

#### Scenario: フッターへのバージョン表示
- **WHEN** ユーザーがアプリケーションの任意の画面を表示したとき
- **THEN** 左側サイドバーの最下部に現在のバージョン情報（例: `v1.0.0` または開発中の場合は `1.0.0-3-g123abcd`）が表示されること

### Requirement: アプリケーション全体のバージョン情報が設定画面に表示されること
システムは、設定画面（Settings）の最下部に「アプリケーション情報」セクションを追加し、現在のバージョン番号を表示しなければならない（SHALL）。

#### Scenario: 設定画面でのバージョン表示
- **WHEN** ユーザーが設定画面（`/settings`）を開いたとき
- **THEN** 設定画面の最下部に「アプリケーション情報」セクションが表示され、そこに現在のバージョン情報が表示されること

### Requirement: 開発環境ではGitタグから動的にバージョンが生成されること
システムは、ローカル開発環境（Vite開発サーバー起動時）において、現在のGitリポジトリの状態（`git describe --tags --always`）から動的にバージョンを生成して表示しなければならない（SHALL）。

#### Scenario: 開発中における動的バージョン表示
- **WHEN** 開発環境でアプリケーションを起動し、Gitリポジトリがタグ `1.0.0` から 3 コミット進んだハッシュ `123abcd` の状態であるとき
- **THEN** 画面上に表示されるバージョンは `1.0.0-3-g123abcd` となること

### Requirement: リリースビルド時にはGitタグ名が埋め込まれること
システムは、GitHubのリリースワークフロー（CI/CD）実行時において、プッシュされたGitタグ名を環境変数 `VITE_APP_VERSION` から読み込み、ビルド成果物に埋め込まなければならない（SHALL）。

#### Scenario: リリースビルドのバージョン埋め込み
- **WHEN** GitHub Actions でタグ `1.2.3` のプッシュによりリリースビルドが実行されるとき
- **THEN** ビルドされた成果物（`dist/`）を動かした際、表示されるバージョンは `1.2.3` となること
