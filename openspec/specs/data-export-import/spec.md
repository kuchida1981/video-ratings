# Data Export / Import Spec

## Purpose

全登録データのバックアップ・環境移行手段を提供する。設定画面のデータ管理セクションから、エクスポート（JSON形式）とインポート（完全置換）を行える。

## Requirements

### Requirement: 全登録データをJSONファイルとしてエクスポートできる
設定画面のデータ管理セクションから、全テーブルのデータをJSONファイルとしてダウンロードできる。出力ファイルには現在のDBスキーマバージョン（Alembicリビジョン）と出力日時を含める。

#### Scenario: エクスポートの実行
- **WHEN** ユーザーが設定画面のデータ管理セクションで「エクスポート」ボタンをクリックする
- **THEN** 全テーブルのデータと `schema_version`・`exported_at` を含むJSONファイルがダウンロードされる

#### Scenario: エクスポートファイルにスキーマバージョンが含まれる
- **WHEN** エクスポートされたJSONファイルを開く
- **THEN** トップレベルに `schema_version` フィールドが存在し、現在のAlembicリビジョン文字列が入っている

---

### Requirement: エクスポートしたJSONファイルから全データをインポートできる
設定画面のデータ管理セクションから、エクスポートと同形式のJSONファイルをアップロードして全データを復元できる。インポートは完全置換（既存データを全削除してから再挿入）で行われる。

#### Scenario: インポート前の確認ダイアログ表示
- **WHEN** ユーザーがJSONファイルを選択してインポートを実行しようとする
- **THEN** 「全データが置き換えられます」という警告とともに確認ダイアログが表示される

#### Scenario: 確認後にインポートが完了する
- **WHEN** ユーザーが確認ダイアログでインポートを承認する
- **THEN** 既存の全データが削除され、ファイルのデータが再挿入される

#### Scenario: インポート成功後の通知
- **WHEN** インポートが正常に完了する
- **THEN** インポート完了のフィードバックが表示される

#### Scenario: schema_version 不一致時はインポートを拒否する
- **WHEN** アップロードされたJSONファイルの `schema_version` が現在のDBリビジョンと異なる
- **THEN** インポートが実行されず、バージョン不一致のエラーメッセージが表示される

#### Scenario: 不正なJSONファイルはエラーになる
- **WHEN** 不正なフォーマットのファイルがアップロードされる
- **THEN** インポートが実行されず、エラーメッセージが表示される

---

### Requirement: 設定画面がカスタム項目とデータ管理の2セクションに整理される
設定ページ（`/settings`）は縦スクロール型のセクション構成になり、「カスタム項目」セクションと「データ管理」セクションを持つ。

#### Scenario: 設定画面の表示
- **WHEN** ユーザーが `/settings` に遷移する
- **THEN** カスタム項目セクションとデータ管理セクションが縦に並んで表示される
