# Performer Management Spec

## Purpose

出演者エンティティの登録・編集・削除と、作品への紐づけ（主演フラグ含む）を管理する機能を提供する。出演者の別名・カスタム項目・タイル表示・作品一覧表示も含む。
## Requirements
### Requirement: 出演者の登録
システムは出演者を個別に登録できなければならない（SHALL）。登録項目は名前（必須）とふりがな（任意）。別名は登録後に追加できる。

#### Scenario: 名前のみで登録できる
- **WHEN** ユーザーが名前のみを入力して登録する
- **THEN** システムは出演者を保存する

#### Scenario: 名前とふりがなを登録できる
- **WHEN** ユーザーが名前とふりがなを入力して登録する
- **THEN** システムは両方の値を保存する

#### Scenario: 名前が空の場合は登録できない
- **WHEN** ユーザーが名前を空のまま登録を実行する
- **THEN** システムはエラーを表示し、保存を行わない

---

### Requirement: 出演者の編集
システムは登録済みの出演者の名前・ふりがな・別名・メモを編集できなければならない（SHALL）。

#### Scenario: 出演者情報を更新できる
- **WHEN** ユーザーが名前またはふりがなを変更して保存する
- **THEN** システムは変更を保存し、その出演者が紐づく全作品に変更が反映される

#### Scenario: 別名を追加できる
- **WHEN** ユーザーが出演者詳細ページで別名の追加フォームに名前を入力して追加する
- **THEN** システムは別名を保存し、別名一覧に表示する

#### Scenario: 別名を編集できる
- **WHEN** ユーザーが登録済みの別名の名前またはふりがなを変更して保存する
- **THEN** システムは変更を保存し、別名一覧に反映する

#### Scenario: 別名を削除できる
- **WHEN** ユーザーが別名の削除ボタンを押す
- **THEN** システムは別名を削除し、別名一覧から消える

### Requirement: 出演者の削除
システムは登録済みの出演者を削除できなければならない（SHALL）。

#### Scenario: 出演者を削除できる
- **WHEN** ユーザーが削除を確認して実行する
- **THEN** システムは出演者を削除し、作品との関連付けも削除する。作品自体は削除しない

---

### Requirement: 作品への出演者紐づけ
システムは1つの作品に対して複数の出演者を紐づけられなければならない（SHALL）。紐づけには主演フラグ（is_main）を設定できる。

#### Scenario: 作品に出演者を追加できる
- **WHEN** ユーザーが作品編集画面で出演者を選択して追加する
- **THEN** システムは出演者を作品に紐づける

#### Scenario: 主演フラグを設定できる
- **WHEN** ユーザーが出演者に主演フラグをつける
- **THEN** システムはその出演者を主演として保存する

#### Scenario: 主演は1作品につき1人まで
- **WHEN** ユーザーが既に主演がいる作品で別の出演者に主演フラグをつける
- **THEN** システムは既存の主演フラグを外し、新しく選択した出演者を主演にする

---

### Requirement: 出演者の作品一覧表示
システムは特定の出演者が出演する全作品をタイルグリッド形式で表示できなければならない（SHALL）。

#### Scenario: 出演者の作品一覧を表示できる
- **WHEN** ユーザーが出演者詳細ページを表示する
- **THEN** システムはその出演者が紐づく全作品を作品タイルのグリッドで表示する

### Requirement: 出演者詳細ページでカスタム項目を表示・編集できる
システムは出演者詳細ページで、出演者用カスタム項目定義が存在する場合にカスタム項目セクションを表示しなければならない（SHALL）。

#### Scenario: 出演者用カスタム項目が存在する場合にセクションが表示される
- **WHEN** ユーザーが出演者詳細ページを表示し、出演者用カスタム項目定義が1件以上登録されている
- **THEN** システムはカスタム項目セクションを表示し、各項目に現在保存されている値（または空）を表示する

#### Scenario: 出演者用カスタム項目が存在しない場合はセクションが表示されない
- **WHEN** ユーザーが出演者詳細ページを表示し、出演者用カスタム項目定義が0件の場合
- **THEN** システムはカスタム項目セクションを表示しない

### Requirement: 出演者一覧ページでタイル表示する
The system MUST display performers as tiles with average work score.
システムは出演者一覧ページで出演者をタイルグリッド形式で表示しなければならない。出演者タイルには名前・出演作品数・スコアを表示する。`GET /performers` レスポンスの各出演者オブジェクトには `avg_work_score`（出演作品の total_score 平均値）を含まなければならない。作品が0件の場合は `avg_work_score = 0.0` とする。

#### Scenario: 出演者タイルの内容
- **WHEN** ユーザーが出演者一覧ページを表示する
- **THEN** 各出演者がタイルとして表示され、名前・出演作品数・スコアがタイル内に含まれる

#### Scenario: 出演者が登録されていない場合
- **WHEN** 出演者が1件も登録されていない状態で出演者一覧ページを表示する
- **THEN** 「出演者が登録されていません」というメッセージを表示する

#### Scenario: avg_work_score が API レスポンスに含まれる
- **WHEN** クライアントが `GET /performers` を呼び出す
- **THEN** 各出演者オブジェクトに `avg_work_score` フィールド（float）が含まれる

#### Scenario: 作品が0件の出演者の avg_work_score
- **WHEN** 作品が紐づいていない出演者が `GET /performers` に含まれる
- **THEN** その出演者の `avg_work_score` は `0.0` である
