# Spec: works-list-display

## Purpose

作品一覧の表示に関する仕様。作品検索APIのレスポンス内容と、フロントエンドの作品一覧テーブルの表示形式を定義する。
## Requirements
### Requirement: 作品一覧は全件返す
作品検索APIは件数制限なしで全件を返さなければならない（SHALL）。ページングパラメータ（page, page_size）は廃止する。

#### Scenario: 全件取得
- **WHEN** フロントエンドが作品検索APIを呼ぶ
- **THEN** 登録されている全作品が返される（20件上限なし）

---

### Requirement: 作品一覧レスポンスに出演者を含む
作品検索APIのレスポンスには、各作品の出演者リスト（id と name のみ）を含まなければならない（SHALL）。

#### Scenario: 出演者ありの作品
- **WHEN** 出演者が登録されている作品が返される
- **THEN** performers フィールドに出演者の id と name のリストが含まれる

#### Scenario: 出演者なしの作品
- **WHEN** 出演者が登録されていない作品が返される
- **THEN** performers フィールドは空リストになる

---

### Requirement: 作品一覧テーブルの列順と内容
作品一覧はタイルグリッド形式とテーブル形式の両方をサポートしなければならない（SHALL）。デフォルト表示はタイルグリッド形式とする。テーブル形式ではサムネ画像を表示せず、タイトルと出演者名を最小列として常に表示する。メーカー・シリーズ・登録日・スコア・タグ・ファイル数・カスタム項目はユーザーが表示列として選択できる。

#### Scenario: 出演者が複数名いる作品の表示（タイル）
- **WHEN** 作品一覧タイルグリッドを表示する
- **THEN** タイル内の出演者欄に出演者名をカンマ区切りで表示する（例: 「田中花子, 山田太郎」）

#### Scenario: 出演者がいない作品の表示（タイル）
- **WHEN** 出演者が0名の作品のタイルを表示する
- **THEN** 出演者欄に「—」を表示する

#### Scenario: テーブル表示時のタイトルと出演者名
- **WHEN** 作品一覧をテーブル形式で表示する
- **THEN** タイトル列と出演者名列が常に表示される

### Requirement: 作品一覧レスポンスにカスタム項目とタグを含む
作品検索APIのレスポンスには、各作品の `custom_fields`（キーバリュー）と `tags`（id・name・category_id・score）を含まなければならない（SHALL）。

#### Scenario: カスタム項目が設定されている作品
- **WHEN** カスタム項目が設定されている作品が返される
- **THEN** `custom_fields` フィールドにキーバリュー形式でカスタム項目の値が含まれる

#### Scenario: タグが付いている作品
- **WHEN** タグが付いている作品が返される
- **THEN** `tags` フィールドに各タグの id・name・category_id・score が含まれる

#### Scenario: カスタム項目・タグが未設定の作品
- **WHEN** カスタム項目もタグも設定されていない作品が返される
- **THEN** `custom_fields` は null、`tags` は空リストになる

### Requirement: 作品一覧のフィルタ全解除はソート条件も対象とする
作品一覧の「フィルタ全解除」操作は、キーワード・メーカー・シリーズ・タグ等のフィルタ条件に加えて、ソート条件（sortBy, sortDesc）もデフォルト値（sortBy="created_at", sortDesc=true）に戻さなければならない（SHALL）。

#### Scenario: ソートのみ変更されている場合もフィルタ全解除ボタンが表示される
- **WHEN** フィルタ条件は全てデフォルトだが、ソート条件がデフォルト値と異なる
- **THEN** フィルタ全解除ボタンが表示される

#### Scenario: フィルタ全解除でソートもリセットされる
- **WHEN** ユーザーがフィルタ全解除ボタンをクリックする
- **THEN** フィルタ条件とともにソート条件も "created_at" 降順にリセットされる
