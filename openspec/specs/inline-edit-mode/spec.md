# Spec: inline-edit-mode

## Purpose

作品一覧・出演者一覧テーブル表示モードにおけるインライン編集機能の仕様。カスタムフィールドのインライン編集、自動保存、エラーフィードバック、編集モード中の行クリック制御を定義する。

## Requirements

### Requirement: 作品一覧テーブルに編集モードを追加する
システムは作品一覧および出演者一覧のテーブル表示モードにおいて、編集モードのON/OFFを切り替えるトグルボタンを提供しなければならない（SHALL）。編集モードは viewMode が table のときのみ利用可能とする。編集モード状態はページ遷移でリセットされる（localStorage に保存しない）。

#### Scenario: 作品一覧テーブル表示時に編集モードトグルが表示される
- **WHEN** ユーザーが作品一覧ページでテーブル表示モードにしている
- **THEN** 編集モードのトグルボタンが表示される

#### Scenario: 出演者一覧テーブル表示時に編集モードトグルが表示される
- **WHEN** ユーザーが出演者一覧ページでテーブル表示モードにしている
- **THEN** 編集モードのトグルボタンが表示される

#### Scenario: タイル表示時に編集モードトグルが表示されない
- **WHEN** ユーザーが作品一覧ページまたは出演者一覧ページでタイル表示モードにしている
- **THEN** 編集モードのトグルボタンは表示されない

#### Scenario: 編集モードをONにする
- **WHEN** ユーザーが編集モードトグルをクリックする
- **THEN** テーブルのカスタムフィールド列がインライン入力欄に変わる

#### Scenario: 編集モードをOFFにする
- **WHEN** ユーザーが編集モード中にトグルをクリックする
- **THEN** テーブルのカスタムフィールド列が読み取り専用表示に戻り、リストが再取得されて最新状態と同期する

#### Scenario: カスタムフィールド列が非表示の場合は編集モードボタンが無効になる
- **WHEN** テーブル表示モードでカスタムフィールド列が1つも表示されていない
- **THEN** 編集モードのトグルボタンは無効化（disabled）される

---

### Requirement: 編集モードでカスタムフィールドをインライン編集できる
編集モードON時、表示中のカスタムフィールド列のセルは型に応じたインライン入力UIに変わらなければならない（SHALL）。作品一覧ではタイトル・出演者・スコア・タグ・ファイル数・登録日、出演者一覧では出演者名・ふりがな・作品数・平均スコア・合計スコア・タグのセルは編集モード中も読み取り専用のままとする。

#### Scenario: text 型のカスタムフィールドがテキスト入力欄になる
- **WHEN** 編集モードON時に text 型のカスタムフィールド列のセルを表示する
- **THEN** セルは `<input type="text">` として表示され、現在の値が入力欄に表示される

#### Scenario: number 型のカスタムフィールドが数値入力欄になる
- **WHEN** 編集モードON時に number 型のカスタムフィールド列のセルを表示する
- **THEN** セルは `<input type="number">` として表示され、現在の値が入力欄に表示される

#### Scenario: date 型のカスタムフィールドが日付入力欄になる
- **WHEN** 編集モードON時に date 型のカスタムフィールド列のセルを表示する
- **THEN** セルは `<input type="date">` として表示され、現在の値が入力欄に表示される

#### Scenario: boolean 型のカスタムフィールドがチェックボックスになる
- **WHEN** 編集モードON時に boolean 型のカスタムフィールド列のセルを表示する
- **THEN** セルは `<input type="checkbox">` として表示され、現在の値が反映される

#### Scenario: 値が未設定のカスタムフィールドのセルは空の入力欄になる
- **WHEN** 編集モードON時にカスタムフィールドの値が null/undefined のエンティティ行を表示する
- **THEN** セルは空の入力欄として表示される（boolean の場合はチェックなし）

---

### Requirement: カスタムフィールドの変更をセル単位で自動保存する
編集モードON時、カスタムフィールドセルのフォーカスが外れた（blur）ときに変更を検出し、変更があった場合のみ対応する API（作品: `PATCH /works/{id}/custom-fields`、出演者: `PATCH /performers/{id}/custom-fields`）で保存しなければならない（SHALL）。boolean 型はチェックボックスの change イベントで即保存する。

#### Scenario: text 型の値を変更して blur で保存する
- **WHEN** ユーザーが text 型のセルの値を変更してフォーカスを外す
- **THEN** システムは対応するカスタムフィールド更新APIを呼び出し、変更されたフィールドの値を保存する

#### Scenario: number 型の値を変更して blur で保存する
- **WHEN** ユーザーが number 型のセルの値を変更してフォーカスを外す
- **THEN** システムは変更された値を数値として保存する

#### Scenario: date 型の値を変更して blur で保存する
- **WHEN** ユーザーが date 型のセルの値を変更してフォーカスを外す
- **THEN** システムは変更された値をISO 8601形式の文字列として保存する

#### Scenario: boolean 型の値を変更して即保存する
- **WHEN** ユーザーが boolean 型のチェックボックスを変更する
- **THEN** システムは即座に値を保存する（blur を待たない）

#### Scenario: 値が変更されていない場合は保存しない
- **WHEN** ユーザーがセルにフォーカスしてから値を変えずにフォーカスを外す
- **THEN** システムはAPI呼び出しを行わない

#### Scenario: 値を空にした場合はフィールドを削除する
- **WHEN** ユーザーが text / number / date 型のセルの値を空にしてフォーカスを外す
- **THEN** システムは `custom_fields` から該当キーを削除する（値を null として送信する）

---

### Requirement: 保存失敗時にエラーフィードバックを表示する
カスタムフィールドの保存APIが失敗した場合、システムは対象セルの枠を赤くハイライトしてエラーを伝えなければならない（SHALL）。保存成功時はUIフィードバックを表示しない。

#### Scenario: 保存が失敗したセルが赤くハイライトされる
- **WHEN** カスタムフィールドの保存APIが失敗する
- **THEN** 対象セルの枠が赤くハイライトされ、値は編集前の状態にロールバックされる

#### Scenario: 保存が成功した場合はフィードバックなし
- **WHEN** カスタムフィールドの保存APIが成功する
- **THEN** UIに追加のフィードバック（トースト、色変化等）は表示されない

---

### Requirement: 編集モード中の行クリック遷移を制御する
編集モードON時、カスタムフィールドセルのクリックは入力フォーカスとして扱い、行全体のクリックによる詳細ページ遷移を発生させてはならない（SHALL NOT）。ただしタイトル列（作品一覧）および出演者名列（出演者一覧）のリンクは維持する。

#### Scenario: 編集モード中にカスタムフィールドセルをクリックしてもページ遷移しない
- **WHEN** 編集モードON時にカスタムフィールドセルをクリックする
- **THEN** セルの入力欄にフォーカスが移り、詳細ページへの遷移は発生しない

#### Scenario: 編集モード中でもタイトル・出演者名リンクから詳細ページに遷移できる
- **WHEN** 編集モードON時にタイトル列（作品一覧）または出演者名列（出演者一覧）のリンクをクリックする
- **THEN** 詳細ページに遷移する

### Requirement: 編集モード時に名前・タイトルセルにGoogle検索リンクを併設する
編集モードが有効なとき、出演者一覧テーブルの出演者名セルおよび作品一覧テーブルのタイトルセルに、詳細ページへのリンクに加えて Google 検索アイコンリンクを表示しなければならない（SHALL）。検索アイコンは詳細ページリンクとは独立した要素として配置し、クリック時に詳細ページへの遷移を発生させてはならない（SHALL NOT）。

#### Scenario: 編集モード時に出演者名セルにリンクと検索アイコンが並ぶ
- **WHEN** 出演者一覧テーブルの編集モードが有効である
- **THEN** 出演者名セルに詳細ページリンクと Google 検索アイコンリンクが横並びで表示される

#### Scenario: 編集モード時にタイトルセルにリンクと検索アイコンが並ぶ
- **WHEN** 作品一覧テーブルの編集モードが有効である
- **THEN** タイトルセルに詳細ページリンクと Google 検索アイコンリンクが横並びで表示される

#### Scenario: 検索アイコンクリック時にページ遷移しない
- **WHEN** 編集モード中に検索アイコンをクリックする
- **THEN** 新しいタブで Google 検索が開き、現在のページからの遷移は発生しない
