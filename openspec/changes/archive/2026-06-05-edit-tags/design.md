## Context

タグ管理画面（`TagsPage.tsx`）でタグの追加・削除は既に実装されているが、編集（名前・スコアの変更）はできない。バックエンドの `PUT /tags/:id` エンドポイントとフロントエンドの `api.tags.update` クライアントは実装済みであるため、変更はフロントエンドUIのみに限定される。

## Goals / Non-Goals

**Goals:**
- タグ一覧から各タグをインライン編集できるようにする
- 編集フォームでタグ名とスコアを変更して保存できる
- キャンセルで編集前の状態に戻れる

**Non-Goals:**
- タグカテゴリ自体の編集機能（カテゴリ名・is_multi_select の編集は既存の `api.tagCategories.update` があるが今回は対象外）
- バックエンド・APIクライアントの変更

## Decisions

### インライン編集フォームを採用する

タグは小さなチップ（pill）UI で表示されており、モーダルを出すほどの情報量ではない。タグを追加する既存のインラインフォームと同じパターンを踏襲し、タグをクリックすると同一箇所に入力フィールドが展開するインライン編集を採用する。

代替案として「編集ダイアログ（モーダル）」も考えられるが、追加フォームとUIパターンが異なると一貫性が損なわれる。

### 編集状態は `editingTagId` ステートで管理する

現在の追加フォームが `tagOpen: number | null`（カテゴリIDを持つ）と同じ方法で、編集中のタグIDを `editingTagId: number | null` で管理する。追加フォームと編集フォームが同時に開かないよう、どちらか一方のみアクティブにする。

## Risks / Trade-offs

- [編集中にページを離れると変更が失われる] → インライン編集の性質上許容する。保存ボタンを明示することで誤操作を防ぐ。
- [追加フォームと編集フォームが競合する] → 編集フォームを開いたときに追加フォームを閉じる（`setTagOpen(null)`）、逆も同様にする。
