## Why

タグ管理UI（TagsPage）でカテゴリとタグの表示順が DB の挿入順に固定されており、ユーザーが論理的な順序に並べ替えられない。並び順を管理できることで、作品・出演者のタグ付け画面でも意図した順序でタグが表示されるようになる。

## What Changes

- `tag_categories` テーブルに `sort_order` カラムを追加し、並び順を永続化する
- `tags` テーブルに `sort_order` カラムを追加し、カテゴリ内のタグ並び順を永続化する
- タグカテゴリ・タグの一覧取得 API が `sort_order` 昇順でソートされた結果を返すようになる
- カテゴリ・タグ新規作成時に末尾の `sort_order` が自動採番される
- 並び順を一括更新する API エンドポイントを追加する
- タグ管理UI でカテゴリ行にドラッグハンドルを追加し、entity_type セクション内でドラッグによる並び替えができるようになる
- タグ管理UI でタグ表示をピル形式から縦リスト形式に変更し、タグ行にドラッグハンドルを追加してカテゴリ内で並び替えができるようになる
- 作品・出演者の詳細画面でのタグ一覧は API の返却順（= sort_order）に自動的に追従する

## Capabilities

### New Capabilities

- `tag-sort-order`: タグカテゴリおよびカテゴリ内タグの並び順を管理する機能。タグ管理UIでのドラッグ操作による並び替えと、その順序の永続化・反映を提供する。

### Modified Capabilities

（既存 spec の要件レベルの変更なし）

## Impact

- **バックエンド**: `TagCategory`・`Tag` モデルへのカラム追加、Alembic マイグレーション、API スキーマ更新、新規 reorder エンドポイント追加
- **フロントエンド**: `@dnd-kit/core` + `@dnd-kit/sortable` の新規依存追加、`TagsPage.tsx` の大幅な書き換え（ドラッグ対応）、フロントエンド型定義の更新
- **API クライアント**: reorder エンドポイント呼び出しの追加
- **作品・出演者詳細ページ**: 変更不要（API の返却順が変わることで自動追従）
