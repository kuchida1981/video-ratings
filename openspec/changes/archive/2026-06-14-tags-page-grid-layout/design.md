## Context

タグ管理画面（`frontend/src/pages/TagsPage.tsx`）は「作品用カテゴリ」「出演者用カテゴリ」の2セクションを持ち、各セクション内でカテゴリを縦1列のアコーディオンリストで表示している。カテゴリ数は現在4つ（各セクション2つ程度）、タグ数はカテゴリあたり3〜10個。画面横幅が活用されていないため余白が大きい。

## Goals / Non-Goals

**Goals:**
- カテゴリを2列グリッドのカードレイアウトで表示し、情報密度を高める
- カード高さはコンテンツに合わせた自然な高さ（`align-items: start`）にする
- カテゴリのDnD並び替えをグリッドレイアウトで維持する

**Non-Goals:**
- タグ内のDnD（縦リスト）の変更
- バックエンドAPIの変更
- カテゴリ・タグのCRUD動作の変更
- レスポンシブ対応の追加（既存のブレークポイント挙動に追従するのみ）

## Decisions

### D1: CSS Grid + `align-items: start`

`grid-cols-2` + `items-start` のTailwindクラスでグリッドを構成し、各カードは自然な高さで表示する。

- **理由**: カード高さを統一すると、タグ数が少ないカードに人工的な余白が生じ、解決しようとしていた問題が別の形で再発する。高さをコンテンツに委ねることが最も情報密度に正直な設計。
- **代替案**: 固定高さ＋内部スクロール → 3〜10タグ程度のスケールでは過剰であり採用しない。

### D2: カテゴリDnDを `rectSortingStrategy` に切り替え

`@dnd-kit/sortable` の `rectSortingStrategy` をカテゴリ並び替えに使用する。

- **理由**: グリッドレイアウトで `verticalListSortingStrategy` を使うと、ドロップ判定が縦方向のみになりグリッド内の左右移動が正しく機能しない。
- **代替案**: カテゴリDnDを廃止してボタン操作に変更 → 使用頻度が低いとはいえ機能を削るのは避ける。

### D3: アコーディオン展開/折りたたみの廃止

グリッドカード形式ではカードの折りたたみは不要。タグリストは常時表示とし、ChevronDown/ChevronRight アイコンとそれに関連する `expanded` stateを削除する。

- **理由**: カードが2列に並ぶと各カードが画面内に収まりやすく、折りたたむ意義が薄れる。常時展開の方がワンアクション少なく操作性が向上する。

## Risks / Trade-offs

- **[カード高さのばらつき]** → `align-items: start` を採用するため、同じグリッド行でも左右のカードが異なる高さになる。視覚的な「揃い感」は失われるが、余白の少なさと情報密度を優先する判断。
- **[カテゴリ数が増えた場合]** → 現状4つ想定の設計だが、カテゴリが増えれば自然にグリッドが縦に伸びるため対応不要。
