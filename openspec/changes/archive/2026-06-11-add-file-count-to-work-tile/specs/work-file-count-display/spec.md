## ADDED Requirements

### Requirement: 作品一覧APIがファイル数を返す
`GET /works` レスポンスの各作品オブジェクトは `file_count` フィールド（整数）を含まなければならない（SHALL）。値はその作品に紐づく `WorkFile` レコードの件数とする。

#### Scenario: ファイルが存在する作品のfile_count
- **WHEN** `GET /works` を呼び出し、ファイルが2件登録されている作品が含まれる場合
- **THEN** その作品の `file_count` は `2` を返す

#### Scenario: ファイルが存在しない作品のfile_count
- **WHEN** `GET /works` を呼び出し、ファイルが0件の作品が含まれる場合
- **THEN** その作品の `file_count` は `0` を返す

### Requirement: 作品タイルにファイル数バッジを表示する
作品タイルは `file_count` が1以上の場合のみファイル数バッジを表示しなければならない（SHALL）。`file_count` が0の場合はバッジを表示してはならない（SHALL NOT）。

#### Scenario: ファイルありの作品タイル表示
- **WHEN** `file_count` が1以上の作品タイルを表示する
- **THEN** ファイルアイコンと件数がタイル下部のスコア左側に表示される

#### Scenario: ファイルなしの作品タイル表示
- **WHEN** `file_count` が0の作品タイルを表示する
- **THEN** ファイル数バッジは表示されない

### Requirement: ファイルなしフィルタ
作品一覧は「ファイルなし」フィルタバッジを提供しなければならない（SHALL）。フィルタが有効な場合、`file_count === 0` の作品のみを表示する。

#### Scenario: ファイルなしフィルタを有効にする
- **WHEN** ユーザーが「ファイルなし」バッジをクリックする
- **THEN** `file_count === 0` の作品のみ表示され、バッジはアクティブ状態になる

#### Scenario: ファイルなしフィルタをリセットする
- **WHEN** ユーザーが「リセット」ボタンをクリックする
- **THEN** 「ファイルなし」フィルタが解除され、すべての作品が表示される

#### Scenario: フィルタが有効な場合のリセットボタン表示
- **WHEN** 「ファイルなし」フィルタが有効な状態
- **THEN** リセットボタンが表示される
